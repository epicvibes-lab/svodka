# Сводка — личный накопительный архив важных новостей

**Сводка** — это автономная лента важных мировых и политических новостей на русском.
Раз в заданный интервал бот сам читает RSS, отбирает действительно значимые события
через модель Claude (Haiku) и **дописывает** их в архив `svodka.json`. Лёгкий дашборд
`svodka.html` показывает накопленную ленту с любого устройства.

Вся инфраструктура бесплатна (GitHub Actions + GitHub Pages). Платите только за вызовы
модели — это центы в месяц.

> **Что нового в версии 2.0.** Раньше `svodka.json` каждый раз перезаписывался: в нём
> оставались только 6 последних новостей, а предыдущие терялись. Теперь это
> **накопительный архив**: при каждом запуске в него добавляются только свежие важные
> новости, которых там ещё не было, а всё прошлое сохраняется.

## Как это работает

Один прогон сборщика (`build.mjs`) проходит по шагам:

1. **Чтение архива.** Загружается текущий `svodka.json`. Из него берётся множество
   уже известных новостей (по ключу `key`). Если файла ещё нет — стартуем с пустого архива.
2. **Сбор RSS.** Читаются ленты из списка `FEEDS`. Отбрасывается всё старше `FRESH_HOURS`
   часов. Внутри одного прогона убираются дубликаты (нормализованный ключ заголовка).
3. **Отсечение уже виденного.** Из свежих заголовков выбрасываются те, чей ключ уже есть
   в архиве. Остаются **только новые** для ленты события.
4. **Отбор главного моделью.** Новые кандидаты отдаются модели Claude. Она по неизменным
   критериям важности выбирает **все** действительно значимые события (а не фиксированные 6).
   Если важного нет — возвращает пустой список.
5. **Дозапись в архив.** Отобранное добавляется **в начало** архива; старые записи остаются.
   Затем архив чистится: удаляются записи старше `KEEP_DAYS` дней и обрезается по
   потолку `MAX_ITEMS`. Результат пишется обратно в `svodka.json`.
6. **Коммит.** GitHub Actions коммитит обновлённый `svodka.json`. Если новых новостей
   не было — файл не меняется и коммита не будет.

Поток данных:

```
RSS-ленты ──▶ свежие за N часов ──▶ минус дубли внутри прогона
                                          │
              архив svodka.json ──────────┤ минус всё, что уже в архиве
                                          ▼
                              новые кандидаты ──▶ Claude (отбор важного)
                                          ▼
                       новые важные ──▶ в начало архива ──▶ чистка ──▶ svodka.json ──▶ дашборд
```

## Что изменилось по сравнению с v1

| | Было (v1) | Стало (v2) |
|---|---|---|
| `svodka.json` | перезаписывался целиком | накопительный архив |
| Сколько добавляется за прогон | ровно 6 новостей | **все** свежие важные, которых ещё не было |
| Прошлые новости | удалялись | сохраняются |
| Повторы между прогонами | возможны | отсекаются по ключу |
| Хранение | только последняя выборка | до `KEEP_DAYS` дней / `MAX_ITEMS` записей |
| Дашборд | плоский список | лента с группировкой по дням |

**Логика отбора (критерии важности) не менялась** — изменился только лимит: вместо
«ровно 6» модель отбирает все значимые события из числа свежих.

## Файлы проекта

- **`svodka.html`** — дашборд. Открываете его и читаете ленту. Группирует новости по дням
  (Сегодня / Вчера / дата), фильтрует по рубрикам, сам подтягивает свежий `svodka.json`.
- **`build.mjs`** — сборщик: RSS → дедупликация → отсев уже виденного → отбор важного
  через Claude → дозапись в архив. Здесь же все настройки (вверху файла).
- **`svodka.json`** — сам архив новостей (его читает дашборд и дополняет сборщик).
- **`package.json`** — манифест проекта, одна зависимость (`rss-parser`).
- **`.github/workflows/update.yml`** — расписание (cron), запуск сборки и автокоммит архива.
- **`.gitignore`** — стандартные исключения Node.

## Формат архива `svodka.json`

```jsonc
{
  "updated": "2026-06-27T07:52:08.798Z",  // когда архив обновляли в последний раз
  "count": 124,                            // сколько всего новостей в архиве
  "items": [
    {
      "key": "сша нанесли удары по ирану у", // ключ дедупликации (первые слова заголовка)
      "title": "США нанесли удары по Ирану у Ормузского пролива",
      "summary": "Одно предложение: что случилось и почему это важно.",
      "category": "Конфликты",               // Мир | Политика | Экономика | Конфликты
      "source": "Интерфакс",                  // издание
      "iso": "2026-06-27T05:10:00.000Z",      // дата публикации (может быть пустой)
      "added": "2026-06-27T07:52:08.798Z"     // когда новость попала в архив
    }
    // … новые записи идут первыми
  ]
}
```

Поля `key`, `iso` и `added` добавлены в v2. Записи из старого формата (без них) сборщик
при первом запуске дополнит автоматически — ничего вручную править не нужно.

## Логика отбора новостей

Модель получает только **свежие, ещё не виденные** заголовки и инструкцию (она же
определяет «тему» ленты):

- оставить **все действительно важные** мировые и политические события: большая политика,
  международные отношения, конфликты, крупная экономика;
- в приоритете — события, которые встречаются в списке несколько раз или явно широко
  освещаются;
- игнорировать развлечения, спорт, светскую хронику и мелочи;
- ничего не выдумывать; если важного нет — вернуть пустой список.

На каждое событие модель возвращает краткий заголовок, ровно одно предложение-суть,
рубрику и источник. Хотите сместить тему (например, добавить технологии или науку) —
правьте текст промпта в функции `summarize` внутри `build.mjs` и/или список `FEEDS`.

## Настройка (один раз)

1. **Создайте репозиторий на GitHub** (рекомендуется *Public* — безлимитные минуты
   Actions и бесплатный хостинг Pages) и загрузите туда все файлы, сохранив структуру
   (`update.yml` должен лежать именно в `.github/workflows/`).

2. **Получите API-ключ Anthropic.** Заведите аккаунт на платформе Claude, пополните
   баланс (хватит $5), выпустите ключ. Постоянного бесплатного тарифа у API нет.

3. **Добавьте ключ в секреты репозитория:**
   *Settings → Secrets and variables → Actions → New repository secret.*
   Имя: `ANTHROPIC_API_KEY`, значение — ваш ключ.

4. **Включите GitHub Pages:**
   *Settings → Pages → Build and deployment → Deploy from a branch → ветка `main`, папка `/ (root)`.*
   Через минуту страница будет доступна по адресу
   `https://<ваш-логин>.github.io/<репозиторий>/svodka.html`.

5. **Запустите первую сборку вручную:**
   вкладка *Actions → workflow «Update svodka» → Run workflow.*
   Архив дополнится свежими новостями, и дашборд оживёт. Дальше сборка идёт сама
   по расписанию.

> В репозитории уже лежит стартовый `svodka.json` с несколькими новостями, так что
> дашборд покажет ленту сразу, ещё до первого автоматического прогона.

## Параметры и как их менять

Все настройки сборщика — вверху `build.mjs`:

| Константа | Назначение | По умолчанию |
|---|---|---|
| `MODEL` | модель Claude для отбора | `claude-haiku-4-5` |
| `MAX_CANDIDATES` | сколько новых кандидатов максимум отдавать модели за прогон | `60` |
| `FRESH_HOURS` | какие новости из RSS считать свежими | `36` |
| `KEEP_DAYS` | сколько дней хранить новость в архиве | `30` |
| `MAX_ITEMS` | жёсткий потолок размера архива | `800` |
| `FEEDS` | список RSS-источников | 4 ленты |

Прочее:

- **Частота обновления** — строка `cron` в `.github/workflows/update.yml`.
  Раз в день: `'0 5 * * *'`. Каждые 6 часов: `'0 */6 * * *'`. Каждый час: `'0 * * * *'`.
  Время в UTC. Архив накопительный, так что частые запуски безопасны — прошлое не теряется.
- **Источники / тема** — массив `FEEDS` и текст промпта в `summarize` (`build.mjs`).
- **Бессрочный архив** — поднимите `KEEP_DAYS` и `MAX_ITEMS` (учтите, что файл будет расти,
  а дашборд грузит его целиком).

## Стоимость

GitHub Actions и Pages — бесплатно (для публичного репозитория). Платный только вызов
модели: один прогон на Haiku ≈ **$0.007**. Раз в день ≈ ~20 центов в месяц; каждый час
≈ ~$5 в месяц. Веб-поиск модели не используется (новости даёт RSS), лишних трат нет.
На практике расход даже ниже: если новых новостей за прогон нет, модель не вызывается совсем.

## Частые вопросы и устранение неполадок

**Дашборд пишет «Файл svodka.json ещё не создан».**
Запустите сборку: *Actions → Update svodka → Run workflow*. Либо проверьте, что GitHub Pages
включён и указывает на корень ветки `main`.

**В Actions ошибка «Нет ANTHROPIC_API_KEY».**
Секрет не добавлен или назван иначе. Проверьте *Settings → Secrets and variables → Actions*,
имя строго `ANTHROPIC_API_KEY`.

**Прогон прошёл, а новых новостей не добавилось.**
Это нормально: значит свежих важных событий, которых ещё нет в архиве, не нашлось. Файл
не меняется, коммита нет. Загляните в лог прогона — там видно, сколько кандидатов было.

**Одна и та же новость попала дважды.**
Дедупликация идёт по первым словам нормализованного заголовка. Если издания дали сильно
разные формулировки, ключи могли разойтись. Можно усложнить `normKey` в `build.mjs`.

**Хочу другую тему/язык/набор рубрик.**
Тема и рубрики задаются промптом в функции `summarize`, источники — массивом `FEEDS`.
Рубрики на дашборде — массив `CATEGORIES` в `svodka.html` (должны совпадать с тем, что
просим у модели).

**Архив стал большим / грузится медленно.**
Уменьшите `KEEP_DAYS` или `MAX_ITEMS`. Сборщик подчистит лишнее при следующем прогоне.

**Запускаю часто и боюсь, что прогоны затрут друг друга.**
В workflow включён `concurrency` (прогоны не накладываются) и `git pull --rebase` перед
push, так что параллельные запуски архив не повредят.
